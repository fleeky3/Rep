## 1. top 명령어
top 명령어는 시스템의 실시간 상태를 모니터링하는 명령어임
CPU, 메모리, 프로세스 등 시스템 자원 사용량을 지속적으로 갱신하며 보여줌

### 특징
- CPU 및 메모리 사용량 실시간 확인
- 프로세스 우선순위, 부하 상태 확인
- k 명령으로 top 화면에서 프로세스 종료 가능

### 주요 옵션
- -u USER : 특정 사용자 프로세스만 표시
- -p PID : 특정 PID만 추적
- -n N : N번 업데이트 후 종료
- k : top 화면에서 특정 PID 종료
- q : top 종료

### 사용 예시
- top -u username 특정 사용자 프로세스만 표시
- top -p 20253064 특정 PID만 추적
- top -n 7 일곱번 업데이트 후 종료

## 2. ps 명령어

ps 명령어는 현재 실행 중인 프로세스 정보를 출력하는 정적 명령어임
한 순간의 프로세스 상태만 보여주며, 시스템 자원 사용량과 프로세스 상태를 확인할 수 있음

### 특징
- 한 순간의 프로세스 상태만 보여줌
- PID, CPU, 메모리 사용량, 상태(STAT) 확인 가능
- grep과 함께 특정 프로세스 검색에 자주 사용됨

### 주요 옵션
- a : 터미널과 무관하게 모든 프로세스 표시
- u : 사용자 중심 정보 표시 
- x : 터미널 없는 프로세스 포함
- -e : 전체 프로세스 출력
- -f : Full format 출력

### 사용 예시
- ps aux 모든 사용자의 프로세스 확인
- ps -ef 전체 프로세스 + 상세 정보 확인
- ps aux | grep python 파이썬 관련 프로세스만 검색

## 3. jobs 명령어

jobs 명령어는 현재 쉘 세션에서 실행 중인 백그라운드 작업 목록을 보여주는 명령어임
주로 쉘 내부에서 실행한 작업 상태를 확인하고, fg나 bg와 함께 사용

### 특징
- 쉘 세션 내부에서 실행한 작업만 표시
- 작업 번호(Jobs ID) 확인 가능
- fg, bg와 함께 사용하여 작업 제어 가능

### 주요 옵션
- jobs : 현재 쉘 세션의 백그라운드 작업 상태 표시
- fg %번호 : 해당 번호 작업을 foreground로 가져오기
- bg %번호 : 해당 번호 작업을 background에서 계속 실행
- -l : PID 포함 자세한 정보 표시
- -p : PID만 표시

### 사용 예시
- jobs 현재 쉘 세션의 모든 백그라운드 작업 확인
- fg %1 1번 작업을 foreground로 가져오기
- bg %2 2번 작업을 background에서 계속 실행
